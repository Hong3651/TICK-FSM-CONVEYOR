# Tick 기반 설비 제어 상태머신 — 컨베이어 분류 시뮬레이터

실제 하드웨어 없이 **콘솔 명령으로 가상 컨베이어 장비를 제어**하는 C 프로그램입니다.
센서·버튼 입력을 받아 상태머신이 판단하고, 모터·실린더·알람 출력을 제어하는
**설비 제어 프로그램의 구조**를 보여주는 것이 목적입니다.

> ⚠️ 현재 개발 진행 중인 포트폴리오 프로젝트입니다. (Day 1 / 7)

---

## 왜 만들었나

로봇/하네스 현장 업무를 하면서 느낀 점을 코드 구조로 옮긴 학습용 프로젝트입니다.
데이터 분석이 아니라 **"입력 → 판단 → 출력"으로 이어지는 제어 구조**를 직접 설계하고
설명할 수 있는 것을 목표로 합니다.

특히 현장에서 자주 마주치는 **"명령을 넣은 시점과 실제로 반영되는 시점의 차이"**를
작게나마 코드로 표현하려 했습니다. (아래 '차별점' 참고)

---

## 핵심 구조 — PLC식 스캔 사이클

실제 PLC가 매 스캔 주기마다 하는 일을 그대로 본떠, **매 tick마다 아래 3단계를 순서대로**
실행합니다. 이 3단 분리가 이 프로젝트의 뼈대입니다.

```
매 tick:
  1) read_inputs()        — 센서/버튼 등 입력을 읽는다
  2) run_state_machine()  — 입력을 바탕으로 상태를 판단·전이한다
  3) write_outputs()      — 모터/실린더/알람 등 출력을 제어한다
```

입력을 읽는 일, 판단하는 일, 출력하는 일을 섞지 않고 분리하는 것이 핵심입니다.

## 상태 (8개)

| 상태 | 의미 |
|------|------|
| `IDLE` | 대기 — 제품 투입을 기다림 |
| `DETECT` | 제품 감지됨 |
| `TRANSFER` | 컨베이어 이송 중 |
| `SETTLING` | 위치센서 감지 후 안정화 대기 (1~2 tick) |
| `SORT` | 분류 동작 (실린더 푸시) |
| `COMPLETE` | 분류 완료 |
| `ERROR` | 오류 (reset 전까지 잠금) |
| `EMERGENCY` | 비상정지 (모든 출력 OFF) |

상태 전이도는 추후 추가 예정입니다.

---

## 차별점 — "명령 시점 vs 반영 시점"

단순 상태머신을 넘어, 현장 감각을 반영한 세 가지 포인트:

1. **명령 요청과 반영의 분리** — `start` 명령은 즉시 동작하지 않고 **다음 tick에 반영**됩니다.
   (실제 설비에서 명령이 스캔 주기에 맞춰 처리되는 것을 모사)
2. **안정화 대기(SETTLING)** — 위치센서 감지 후 바로 분류하지 않고 **1~2 tick 안정화**를 기다립니다.
3. **타임아웃 에러** — 위치센서가 정해진 tick 안에 들어오지 않으면 **ERROR 상태**로 진입합니다.

## 명령어 (예정)

| 명령 | 동작 |
|------|------|
| `product_on` | 제품 투입 (입구 센서 ON) |
| `start` | 컨베이어 가동 요청 (다음 tick에 반영) |
| `tick` | 스캔 사이클 1회 진행 |
| `status` | 현재 장비 상태 출력 |
| `emergency` | 비상정지 |
| `reset` | 초기 상태로 복귀 |
| `exit` | 프로그램 종료 |
